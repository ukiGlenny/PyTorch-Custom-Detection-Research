# PyTorch-Custom-Detection-Research


# Сравнение архитектур SSD и DETR на датасете VisDrone

## Цель работы

Сравнить две архитектуры для задачи детекции объектов:

- **SSD (Single Shot MultiBox Detector)** — классический одностадийный детектор  
- **DETR (Detection Transformer)** — современный трансформерный подход  

**Датасет:** VisDrone (подвыборка из `VisDrone2019-DET`)  
**Классы:** 4 категории (пешеходы, автомобили, велосипеды, люди; маппинг `{3:0, 4:1, 5:2, 8:3}`).

---

## 🛠 Импорт и загрузка данных

- **Фреймворк:** PyTorch, TorchVision  
- **Вспомогательные библиотеки:** OpenCV, Matplotlib, tqdm, KaggleHub  
- **Загрузка:** данные загружены через `kagglehub.dataset_download`

---

## 🧩 Предобработка данных (`VisDroneDataset`)

| Тип выборки | Аугментации |
|-------------|--------------|
| **Train**   | Resize(640,640), RandomHorizontalFlip(0.5), ColorJitter, Normalize(ImageNet) |
| **Val**      | Resize(640,640), Normalize(ImageNet) |

- **Размер выборки:** 2000 изображений для тренировки, 1070 для валидации  
- **Batch size:** 4  
- **Collate function:** возвращает батч изображений и список таргетов.

---

## Архитектуры моделей

### A. Модель SSD

- **Бэкбон:** ResNet18 (предобученный, `weights='DEFAULT'`), обрезанный до последнего сверточного слоя  
- **Дополнительные свертки:** `Conv2d` с шагом 2 (4 уровня признаков)  
- **Выход:** отдельные свертки для локализации (`loc`) и классификации (`conf`)  
- **Общее количество параметров:** **15 940 596**

### B. Модель DETR

- **Бэкбон:** ResNet18  
- **Трансформер:** `nn.Transformer` (3 энкодера, 3 декодера, `d_model=256`, 8 голов)  
- **Ключевая особенность:** `num_queries=50` — объектные запросы  
- **Выход:** `bbox_coords` (сигмоидная нормализация) и `class_logits`  
- **Общее количество параметров:** **20 480 841**

---

## ⚙ Функции потерь

### SSD Loss

- **Локализация:** Smooth L1 Loss (только по положительным якорям)  
- **Классификация:** Cross-Entropy Loss с балансировкой положительных/отрицательных якорей (1:3)  
- **Якоря (`generate_ssd_anchors`):**
  - 4 уровня признаков: 40×40, 20×20, 10×10, 5×5  
  - 4 или 6 якорей на ячейку  
  - **Всего 9500 якорей**  
  - Размеры: `[24, 48, 96, 128]` (нормализованы относительно 640)

### DETR Loss

- Упрощённая версия (без венгерского алгоритма, используется прямое сопоставление `[:n]`)  
- **Локализация:** L1 Loss  
- **Классификация:** Cross-Entropy Loss

---

##  Обучение

- **Оптимизатор:** AdamW (`lr=1e-4`)  
- **Планировщик:** CosineAnnealingLR  
- **Градиентная аккумуляция:** 4 шага → эффективный батч = 16  
- **Эпохи:** 10  

### Динамика потерь

- **SSD:** Loss снижается с ~5.40 до ~3.76; валидационная плато около 4.1  
- **DETR:** Loss снижается с ~1.02 до ~0.83; валидация ≈0.85 — значительно ниже, чем у SSD

---

##  Результаты и сравнение

| Модель | Лучший Val Loss (Epoch) | Финальный Val Loss | mAP@0.5:0.95 |
|--------|------------------------|---------------------|---------------|
| **SSD**  | 4.0878 (Epoch 10)       | 4.0878              | **0.3668**     |
| **DETR** | **0.8442** (Epoch 7)     | 0.8551              | **0.7174**     |

### Ключевые выводы

- **Победитель по Loss:** DETR — улучшение на **79.35%**  
- **Победитель по mAP:** DETR — лучше на **35.06%**  
- DETR показывает гораздо более высокое качество детекции.

---

##  Итоги

1. **Лидер качества:** DETR значительно превосходит SSD по всем метрикам (Loss и mAP) на задаче детекции малых объектов с дрона (VisDrone).  
2. **Сложность обучения:** Несмотря на большее количество параметров (20M против 15M), DETR обучается стабильнее и достигает низкого значения ошибки быстрее.  
3. **Рекомендация:** Использовать **DETR**, когда критична точность обнаружения (особенно мелких объектов), ценой большего потребления памяти и времени инференса.  
4. **SSD:** Классическая архитектура, подходит для высокоскоростного инференса на простых сценах, но проигрывает трансформерам в сложных условиях (перспектива сверху, перекрытия).
